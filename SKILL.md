# 知识漫游

用户输入一个概念词，生成一篇完整的深度知识漫游文章，自动保存到 Obsidian。

## 触发方式

```
/知识漫游                  # 打开状态面板（消化/新概念/待建清单）
/知识漫游 概念词           # 直接生成新文章
/知识漫游 待建清单          # 查看所有待建节点
/知识漫游 更新 概念词       # 清理原文中已建词的（待建）标注
```

生成文章后，用户可直接回复：
- 发送问题/想法 → 进入探讨对话
- 「跳过」→ 标记该文章已消化，不再提醒
- 「下次」→ 标记该文章待消化，下次生成新文章时提醒

探讨过程中，当用户发出结束信号（见下方结束信号词表），执行【回写消化板块】。

**结束信号词表**（出现以下任意表达即触发回写，不限于此列表，根据语境判断）：

| 类型 | 示例 |
|------|------|
| 明确结束 | "好了就这样"、"完事了"、"就到这里" |
| 确认完成 | "OK"、"明白了"、"懂了" 出现在对话尾部 |
| 转移话题 | "我们聊下一个"、"换个话题" |

---

## 指令路由

收到指令后，先判断是否带参数：

- **无参数**（用户只输入 `/知识漫游`）→ 执行【启动面板】
- **有参数**，参数是 `待建清单` → 执行【查看待建清单】
- **有参数**，参数以 `更新` 开头 → 执行【更新原文标注】
- **有参数**，其他 → 执行【生成新文章】

有参数时直接执行对应流程，不做任何额外检查。

---

## 【启动面板】流程

仅在无参数时触发。

1. 读取 `~/.claude/skills/zhishi-manyou/config.sh` 获取 `{vault}`
2. 并行读取以下两个数据：
   - 清单「待消化」区块中 `- [ ]` 的数量 → 记为 N消化
   - 清单「待处理」区块中 `- [ ]` 的数量 → 记为 N待建
3. 显示状态面板：

```
📚 知识漫游
📌 待消化 N消化 篇　　📋 待建 N待建 个

想做什么？
1. 消化
2. 生成新概念
3. 查看待建清单
```

4. 等待用户回复：
   - 回复 `1` 或「消化」→ 列出所有待消化文章名，等用户选择，进入探讨对话
   - 回复 `2` 或概念词 → 执行【生成新文章】
   - 回复 `3` 或「待建」→ 执行【查看待建清单】

## 【生成新文章】流程

### 第1步：加载配置

运行以下命令，检查配置文件是否存在：

```bash
cat ~/.claude/skills/zhishi-manyou/config.sh 2>/dev/null
```

**如果配置文件不存在或为空：**

向用户提问：

```
🗂️ 首次运行，需要配置一下。

你的 Obsidian 库路径是什么？
（例如：/Users/yourname/Documents/Obsidian Vault）
```

等待用户输入路径，然后运行以下命令写入配置：

```bash
echo 'OBSIDIAN_VAULT="{用户输入的路径}"' > ~/.claude/skills/zhishi-manyou/config.sh
```

然后告知用户：
```
✅ 配置已保存。以后不需要再配置了。
```

**如果配置文件存在：**

读取 `OBSIDIAN_VAULT` 的值，后续所有路径都使用这个变量。

配置完成后，`OBSIDIAN_VAULT` 的值记为 `{vault}`，后续路径：
- 概念库：`{vault}/6.Wiki(概念库)/`
- 知识漫游：`{vault}/3.Sources(原材料)/知识漫游/`
- 待建清单：`{vault}/3.Sources(原材料)/知识漫游/00-待建清单.md`

### 第2步：检测 hook 配置

读取 `~/.claude/settings.json`，检查是否已包含 `update-links.sh` 的 hook 配置。

**如果未配置：**

读取 `~/.claude/settings.json` 当前内容，将以下 hook 合并写入（保留原有内容）：

```json
"hooks": {
  "PostToolUse": [
    {
      "matcher": "Write",
      "hooks": [
        {
          "type": "command",
          "command": "bash ~/.claude/skills/zhishi-manyou/update-links.sh"
        }
      ]
    }
  ]
}
```

然后告知用户：
```
⚙️ 已自动配置双链更新 hook。
```

**如果已配置：** 跳过。

### 第3步：确保输出目录存在

```bash
mkdir -p "{vault}/3.Sources(原材料)/知识漫游"
```

### 第4步：扫描已有概念词

```bash
ls "{vault}/6.Wiki(概念库)/" 2>/dev/null | sed 's/\.md$//'
ls "{vault}/3.Sources(原材料)/知识漫游/" 2>/dev/null | sed 's/\.md$//' | grep -v '^00-'
```

将合并结果记录为"已有概念词列表"。

### 第5步：生成知识漫游文章

根据用户输入的概念词，按照 `article-template.md` 的结构生成完整文章。

**双链处理规则：**
- 已在"已有概念词列表"中 → `[[概念词]]`
- 不在列表中 → `[[概念词]]（待建，一句话简单定义，15字以内）`
- 待建词的定义要让读者一眼明白是什么，不需要跳出去查

### 第6步：保存文章到 Obsidian

用 Write 工具将文章写入：

```
{vault}/3.Sources(原材料)/知识漫游/{概念词}.md
```

### 第7步：更新待建清单

读取 `{vault}/3.Sources(原材料)/知识漫游/00-待建清单.md`（不存在则创建），执行以下操作：
1. 提取刚生成文章中所有 `（待建）` 的概念词
2. 已在知识漫游文件夹中存在的词 → 在清单中标记为 `- [x]`
3. 新出现的待建词 → 追加 `- [ ] 概念词（来源：{当前概念词}）`
4. 在清单的「待消化」区块追加当前概念词：`- [ ] {概念词}（待消化）`

清单格式：

```markdown
# 待建清单

> 每次生成知识漫游后自动更新。输入 `/知识漫游 概念词` 即可深入任意待建节点。

## 待处理
- [ ] 工作记忆（来源：元认知倾向）

## 已完成
- [x] 元认知倾向

## 待消化
- [ ] 贝叶斯（待消化）
- [x] 元方法论（已消化）
- [-] 执行功能（已跳过）
```

将更新后的清单用 Write 工具写入：

```
{vault}/3.Sources(原材料)/知识漫游/00-待建清单.md
```

### 第8步：检查待消化提醒

⚠️ **此步骤不可跳过，必须在第9步之前完成。**

读取清单中「待消化」区块，统计 `- [ ]` 条目数量（不含刚生成的这篇）。

**如果有待消化文章：** 在完成提醒前先显示：

```
📌 你还有 N 篇文章待消化：概念词A、概念词B……需要现在处理吗？
```

### 第9步：显示完成提醒

显示：

```
✅ 已保存至 Obsidian：知识漫游/{概念词}.md

有什么看完想聊的直接发给我。没有问题回复「跳过」，下次再看回复「下次」。
```

收到用户回复后：
- 收到「跳过」→ 将清单中 `- [ ] {概念词}（待消化）` 改为 `- [-] {概念词}（已跳过）`，回复"已标记，不再提醒。"
- 收到「下次」→ 保持 `- [ ]` 不变，回复"好，下次提醒你。"
- 收到其他内容 → 进入探讨对话，直到用户发出结束信号后执行【回写消化板块】

## 【查看待建清单】流程

1. 读取 `~/.claude/skills/zhishi-manyou/config.sh` 获取 `{vault}`
2. 读取并展示 `{vault}/3.Sources(原材料)/知识漫游/00-待建清单.md` 的内容
3. 提示：

```
以上是你的待建概念清单。输入 /知识漫游 概念词 开始深入任意一个。
```

## 【更新原文标注】流程

参数格式：`/知识漫游 更新 概念词`

1. 读取 `~/.claude/skills/zhishi-manyou/config.sh` 获取 `{vault}`
2. 重新扫描"已有概念词列表"（同第4步）
3. 读取指定的知识漫游文章
4. 将文章中所有 `[[已存在概念词]]（待建，...）` 和 `[[已存在概念词]]（待建）` 替换为 `[[已存在概念词]]`
5. 用 Write 工具写回原文件
6. 显示：

```
✅ 已更新：知识漫游/{概念词}.md
   清理了 N 个已建概念的待建标注。
```

---

## 【回写消化板块】流程

⚠️ **探讨结束信号出现后必须立即执行此流程，不可省略。**

当用户发出结束信号（参考触发方式中的结束信号词表）后触发。

1. 读取 `~/.claude/skills/zhishi-manyou/config.sh` 获取 `{vault}`
2. 读取对应文章 `{vault}/3.Sources(原材料)/知识漫游/{概念词}.md`
3. 将本次对话中所有探讨内容整理成结构化文字，追加到文章末尾：

```markdown
## 消化

> {消化日期，格式 YYYY-MM-DD}

### 我的问题

{用户在对话中提出的疑问，逐条列出}

### 探讨过程

{对话中关键的论点、转折、澄清，写成连贯段落，去掉来回确认的废话，保留实质内容}

### 落地的理解

{探讨后用户形成的新认知、修正的旧认知、或者留下的新疑问。用第一人称写，代表用户视角。}
```

4. 用 Write 工具将更新后的完整文章写回原路径
5. ⚠️ **必须同步更新清单**：将清单中 `- [ ] {概念词}（待消化）` 改为 `- [x] {概念词}（已消化）`，写回清单文件
6. 显示：

```
✅ 消化内容已追加到：知识漫游/{概念词}.md
```

## Common Mistakes

| 错误 | 后果 | 纠正 |
|------|------|------|
| 第8步直接跳到第9步 | 待消化提醒永远不出现，堆积越来越多 | 每次生成后必须先读清单「待消化」区块再显示完成提醒 |
| 探讨结束后忘记回写消化板块 | 对话内容消失，文章没有消化记录 | 出现结束信号词后立即触发【回写消化板块】，不等用户提醒 |
| 回写消化后忘记更新清单状态 | 已消化的文章仍显示为待消化，下次继续提醒 | 回写文章和更新清单必须作为一个原子操作，缺一不可 |
| 把"OK"/"明白了"误判为结束信号 | 对话中途触发回写，内容不完整 | 结合上下文判断，只在对话明显收尾时触发，单独的确认词不算 |

## Hook 脚本说明

`update-links.sh` 在每次 Write 工具写入后自动触发，读取 `config.sh` 获取路径，扫描知识漫游文件夹，将已建文件对应的 `（待建）` 标注自动清理。
