# 知识漫游

用户输入一个概念词，生成一篇完整的深度知识漫游文章，自动保存到 Obsidian。

## 触发方式

```
/知识漫游 概念词           # 生成新文章
/知识漫游 待建清单          # 查看所有待建节点
/知识漫游 更新 概念词       # 清理原文中已建词的（待建）标注
```

---

## 指令路由

收到指令后，先判断类型：

- 参数是 `待建清单` → 执行【查看待建清单】
- 参数以 `更新` 开头 → 执行【更新原文标注】
- 其他 → 执行【生成新文章】

---

## 【生成新文章】流程

### 第1步：加载配置

运行以下命令，检查配置文件是否存在：

```bash
cat ~/.claude/skills/zhishi-manyou/config.sh 2>/dev/null
```

**如果配置文件不存在或为空：**

向用户提问：

```
🗂️ 首次运行，需要配置一下。

你的 Obsidian 库路径是什么？
（例如：/Users/yourname/Documents/Obsidian Vault）
```

等待用户输入路径，然后运行以下命令写入配置：

```bash
echo 'OBSIDIAN_VAULT="{用户输入的路径}"' > ~/.claude/skills/zhishi-manyou/config.sh
```

然后告知用户：
```
✅ 配置已保存。以后不需要再配置了。
```

**如果配置文件存在：**

读取 `OBSIDIAN_VAULT` 的值，后续所有路径都使用这个变量。

配置完成后，`OBSIDIAN_VAULT` 的值记为 `{vault}`，后续路径：
- 概念库：`{vault}/6.Wiki(概念库)/`
- 知识漫游：`{vault}/3.Sources(原材料)/知识漫游/`
- 待建清单：`{vault}/3.Sources(原材料)/知识漫游/00-待建清单.md`

### 第2步：检测 hook 配置

读取 `~/.claude/settings.json`，检查是否已包含 `update-links.sh` 的 hook 配置。

**如果未配置：**

读取 `~/.claude/settings.json` 当前内容，将以下 hook 合并写入（保留原有内容）：

```json
"hooks": {
  "PostToolUse": [
    {
      "matcher": "Write",
      "hooks": [
        {
          "type": "command",
          "command": "bash ~/.claude/skills/zhishi-manyou/update-links.sh"
        }
      ]
    }
  ]
}
```

然后告知用户：
```
⚙️ 已自动配置双链更新 hook。
```

**如果已配置：** 跳过。

### 第3步：确保输出目录存在

```bash
mkdir -p "{vault}/3.Sources(原材料)/知识漫游"
```

### 第4步：扫描已有概念词

```bash
ls "{vault}/6.Wiki(概念库)/" 2>/dev/null | sed 's/\.md$//'
ls "{vault}/3.Sources(原材料)/知识漫游/" 2>/dev/null | sed 's/\.md$//' | grep -v '^00-'
```

将合并结果记录为"已有概念词列表"。

### 第5步：生成知识漫游文章

根据用户输入的概念词，生成完整文章。

**双链处理规则：**
- 已在"已有概念词列表"中 → `[[概念词]]`
- 不在列表中 → `[[概念词]]（待建，一句话简单定义，15字以内）`
- 待建词的定义要让读者一眼明白是什么，不需要跳出去查

**文章结构：**

```
---
概念: {概念词}
学习日期: {今天日期，格式 YYYY-MM-DD}
来源学科: [{涉及的主要学科，逗号分隔}]
---

# {概念词}

## 一句话定义

{用普通人能说出口的语言，不是百科式定义。读完这句话，能直接跟别人解释这个词。}

## 来源与演变

{这个词从哪里来：提出者、时间、背景。中间经历了什么演变。现在的理解和最初有什么不同。写成连贯段落，不要写成列表。}

## 不同学科怎么看

根据概念特性灵活选择3-5个最相关的学科，每个学科给出独立视角，末尾标注延伸入口。格式：

**学科名**
{该学科的解读，3-5句话，有实质内容，不凑字数}
↳ 可继续深入：[[相关概念]] 或 [[相关概念]]（待建，定义）

（根据概念特性调整学科范围，可包括：认知心理学、神经科学、哲学西方、东方哲学、社会学、经济学、教育学等）

## 争议在哪里

{支持者的核心论点。质疑者的核心论点。争议至今未解决的部分。写成连贯段落。}

## 边界与对立面

{容易混淆的概念是什么，区别在哪里。对立面或反面是什么。这个概念的局限和适用边界。}

## 知识地图

**上游**
{这个概念从哪些更基础的概念发展而来，带双链}

**同层**
{和它平行的兄弟概念有哪些，带双链}

**下游**
{它可以延伸出哪些方向，带双链}

## 延伸节点

列出5-8个值得深入探索的方向：

| 节点 | 为什么值得深入 | 状态 |
|------|-------------|------|
| [[概念词]] | {一句话说明} | 已有 |
| [[概念词]]（待建，定义） | {一句话说明} | 待建 |

## 和你的关联

**短视频创作**
{这个概念如何用于选题、文案、结构。必须具体，举出可直接用的场景。}

**课程教学**
{如何用这个概念帮助学员理解或解决问题。结合用户的知识输出场景。}

**个人成长 / 工作效率**
{这个概念对个人决策、认知提升、日常工作的具体启发。}

---

## 概念标签

{列出文章中所有双链。已有的写 [[概念词]]，待建的写 [[概念词]]（待建，定义）}
```

**写作约束：**
- 风格：严肃但不学术，像博学的朋友在讲，不是教科书
- 每个板块必须有实质内容，不能为填结构而凑字数
- "和你的关联"必须具体能直接用
- 待建词定义控制在15字以内

### 第6步：保存文章到 Obsidian

用 Write 工具将文章写入：

```
{vault}/3.Sources(原材料)/知识漫游/{概念词}.md
```

### 第7步：更新待建清单

读取 `{vault}/3.Sources(原材料)/知识漫游/00-待建清单.md`（不存在则创建），执行以下操作：
1. 提取刚生成文章中所有 `（待建）` 的概念词
2. 已在知识漫游文件夹中存在的词 → 在清单中标记为 `- [x]`
3. 新出现的待建词 → 追加 `- [ ] 概念词（来源：{当前概念词}）`

清单格式：

```markdown
# 待建清单

> 每次生成知识漫游后自动更新。输入 `/知识漫游 概念词` 即可深入任意待建节点。

## 待处理
- [ ] 工作记忆（来源：元认知倾向）

## 已完成
- [x] 元认知倾向
```

将更新后的清单用 Write 工具写入：

```
{vault}/3.Sources(原材料)/知识漫游/00-待建清单.md
```

### 第8步：显示完成提醒

统计待处理清单中未勾选的数量，显示：

```
✅ 已保存至 Obsidian：知识漫游/{概念词}.md
📋 待建清单已更新，共有 N 个待建概念。
   输入 /知识漫游 待建清单 查看，或直接输入 /知识漫游 概念词 继续深入。
```

---

## 【查看待建清单】流程

1. 读取 `~/.claude/skills/zhishi-manyou/config.sh` 获取 `{vault}`
2. 读取并展示 `{vault}/3.Sources(原材料)/知识漫游/00-待建清单.md` 的内容
3. 提示：

```
以上是你的待建概念清单。输入 /知识漫游 概念词 开始深入任意一个。
```

---

## 【更新原文标注】流程

参数格式：`/知识漫游 更新 概念词`

1. 读取 `~/.claude/skills/zhishi-manyou/config.sh` 获取 `{vault}`
2. 重新扫描"已有概念词列表"（同第4步）
3. 读取指定的知识漫游文章
4. 将文章中所有 `[[已存在概念词]]（待建，...）` 和 `[[已存在概念词]]（待建）` 替换为 `[[已存在概念词]]`
5. 用 Write 工具写回原文件
6. 显示：

```
✅ 已更新：知识漫游/{概念词}.md
   清理了 N 个已建概念的待建标注。
```

---

## Hook 脚本说明

`update-links.sh` 在每次 Write 工具写入后自动触发，读取 `config.sh` 获取路径，扫描知识漫游文件夹，将已建文件对应的 `（待建）` 标注自动清理。
